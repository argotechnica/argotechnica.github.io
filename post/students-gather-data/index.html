A student's guide to collecting web data for free using cloud computing

Fri, Apr 7, 2017 technical, data, tools

Background

At the Oxford Internet Institute, Master’s students learn techniques of data wrangling and analysis using Python 3. Python really shines at being able to gather data from diverse sources, both structured and unstructured—for example via an Application Programming Interface (API), or by scraping web pages directly. In classes, at hackathons, and more, the standard setup for this kind of data wrangling is to run Jupyter or Spyder on our local laptops. However, for research purposes, it’s often best to gather data over time rather than only at a single point in time. For such needs, running a Python script from one’s laptop becomes prohibitively inefficient, and worse: few laptops are on, and online, 24⁄7, meaning that some data cannot be captured.

The solution is to run a recurring (i.e. scheduled) or continuous script on some kind of always-on server or service. Many possible approaches exist for such a solution; this guide constrains these possibilities in two ways: first, to focus only on server solutions rather than specialized compute or task services such as Amazon Web Services (AWS) Lambda; and second, to solve for cost, i.e. to use only services that can be obtained for free.

One free option for running server tasks available at University of Oxford is the IT Services GNU/Linux service. This service enables any student to sign in to a larger, shared server for running tasks or manipulating files. However, “resource-intensive tasks” are forbidden, and the environment itself—which runs only Python 2.7—does not allow for customization of installed software or Python packages. A lot of raw data gathering can still be accomplished in this environment with curl and crontab, but a more robust server environment is preferable in most cases.
